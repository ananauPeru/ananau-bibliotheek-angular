Step 1 - Clone the Repo

If you’ve successfully completed the previous guides, you should have access to Añañau's GitHub repositories. Clone the educational app locally to a desired location on your computer. This allows you to review the implemented functionalities and make future changes.

Step 2 - Use the Back-End and MySQL Database

This app uses the same back-end and MySQL setup as the other previously implemented projects, so check the earlier guides for instructions on running them locally.

Step 3 - Make Your Changes (Best Practices)

A - Overview of Implemented Functionalities
  • Students: List-overview + Create + Delete
  • Exercises: List-overview + Create + Edit + Delete
  • Making exercises: A student can complete a minimum of one and a maximum of all exercises at a given moment. At the end of the session, the student will see how many questions they answered correctly.
  • Results: List-overview + Delete
B - Expanding Exercises

Currently, the logic for exercises is tailored to the specific structure requested by the customer. If they wish to introduce more variables or change how exercises are modeled (e.g., more images, sounds, no option buttons, etc.), this can be done fairly easily. Exercises inherit from a parent class that contains common variables (e.g., question, ID). To implement a new type of exercise, create a new object that inherits from the parent and add any specific properties required.

After updating the controller, service, and other layers to work with the new exercise via a new DTO, you’ll need to reflect these variables in the relevant component to ensure they’re visible on the screen. As a developer, you’ll likely know how to manage this process, but documenting how to expand the exercises was part of my task—so let’s keep it clear. Also, don’t hesitate to check resources like Stack Overflow, YouTube tutorials, etc.—this takes part of a developer's workflow. Good luck!
